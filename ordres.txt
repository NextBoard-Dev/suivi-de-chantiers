REPRISE PROPRE - SUIVI DE CHANTIERS

Contexte projet
- Application web locale "Suivi de Chantiers" avec:
- Tableau maitre
- Vue projet/taches
- Gantt maitre/projet
- Modale saisie heures reelles
- Modale configuration (statuts, prestataires, techniciens internes par site, utilisateurs)

Objectif global
- Continuer les evolutions UI/UX sans regression metier.
- Fiabiliser la modale heures reelles et ses indicateurs.
- Maintenir une coherence visuelle stable entre composants.
- Preserver l'integrite des donnees JSON et des calculs.

Regles Snapshot (standard)
- Snapshot obligatoire avant chaque patch.
- Dossier: snapshots_ok/
- Nom: YYYYMMDD_HHMMSS_CONTEXTE_COURT
- Fichiers minimum: app.js, index.html, style.css

Regles Snapshot Majeur (obligatoires)
- Utiliser un snapshot majeur pour:
- rollback
- migration de donnees
- modifications multi-fichiers sensibles
- changements modale heures + calculs/KPI
- tout lot a risque inter-modules
- Nom majeur obligatoire:
  YYYYMMDD_HHMMSS_⚠️_SNAP_MAJEUR_CONTEXTE_CLAIR
- Fichiers obligatoires snapshot majeur:
  app.js, index.html, style.css, print.css, suivi_chantiers_backup.json
- Avant rollback:
  1) snapshot majeur de l'etat courant
  2) restauration exacte du snapshot demande
- En cas de doute: traiter comme MAJEUR.

Methodologie d'execution
1. Diagnostic cible avant patch:
- source exacte du probleme
- fichiers/lignes impactes
- conflits CSS/JS potentiels
2. Un lot = une cause racine.
3. Eviter les patchs empiles contradictoires.
4. Verification post-patch:
- run_tests.bat obligatoire
- controle visuel des zones impactees uniquement
5. Si non conforme:
- rollback propre vers snapshot valide
- nouveau diagnostic, sans tuning aveugle

Priorites fonctionnelles
1. Modale heures:
- parcours heures manquantes fiable
- selection jour/carte coherente
- KPI instantanes a jour
- prise en compte techniciens internes par site/tache
2. Integrite des heures:
- heures saisies reelles prioritaire
- coherence tableaux/Gantt/graphes/exports
3. UI:
- composants homogenes
- lisibilite elevee
- interactions stables (dropdown, toggles, scroll)
4. Configuration:
- techniciens internes par site robustes
- impact limite a la saisie d'heures

Format de fin de lot (obligatoire)
- Snapshot: <chemin>
- Changements: <3-6 points max>
- Tests: run_tests.bat = <OK/KO>
- Phrase de commit: type(scope): description
